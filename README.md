## 🚀 Быстрый старт

### 1️⃣ Клонирование или копирование

```bash
git clone https://github.com/MrModel228/TeamAi.git
cd TeamAi
```

### 2️⃣ Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3️⃣ Настройка API ключа

Создай файл `.env` в корне проекта:

```bash
# Linux/Mac
touch .env

# Windows (PowerShell)
ni .env
```

Заполни его содержимым:

```env
FM_TOKEN=твой_ключ_cloud_ru
FM_BASE_URL=https://foundation-models.api.cloud.ru/v1
MODEL_NAME=zai-org/GLM-4.7
```

> 🔑 Где взять ключ: cloud.ru → AI Models → Создать API ключ

### 4️⃣ Запуск

```bash
# Веб-интерфейс (рекомендуется)
python web.py
# → Открой http://localhost:5000

# Или консольная версия
python run.py
```

## ⚙️ Настройка

### Файл .env — все параметры

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| FM_TOKEN | (обязательно) | Твой API ключ от cloud.ru |
| FM_BASE_URL | https://foundation-models.api.cloud.ru/v1 | Базовый URL провайдера |
| MODEL_NAME | zai-org/GLM-4.7 | Название модели |

### Проверка подключения

После запуска web.py или run.py ты увидишь:

```
✅ Подключено к: https://foundation-models.api.cloud.ru/v1
✅ Модель: zai-org/GLM-4.7
```

Если ошибка — проверь:
- API ключ в .env
- Доступ к интернету
- Активность подписки на cloud.ru

## 👥 Команда агентов

### 🎯 Оркестратор (Team Lead)
**Всегда подключается первым**

Анализирует задачу и автоматически выбирает, каких агентов подключить.

**Примеры решений:**
```
"Напиши функцию" → 💻 Кодер
"Проверь код" → 🔍 Ревьюер  
"Создай тесты" → 🧪 Тестировщик
"Напиши README" → 📝 Документатор
"Полный цикл" → 💻 + 🔍 + 🧪 + 📝
```

### 💻 Кодер (Senior Developer)
- Пишет чистый, рабочий код
- Следует best practices
- Добавляет комментарии
- Использует современные подходы

### 🔍 Ревьюер (Code Reviewer)
- Ищет баги и уязвимости
- Проверяет производительность
- Предлагает улучшения
- Оценивает читаемость

### 🧪 Тестировщик (QA Engineer)
- Пишет unit-тесты и интеграционные
- Находит edge cases
- Проверяет покрытие кода
- Думает как "злоумышленник"

### 📝 Документатор (Technical Writer)
- Пишет README и документацию
- Создаёт комментарии в коде
- Делает инструкции по установке
- Описывает API endpoints

## ❓ FAQ

**В: Как получить API ключ cloud.ru?**  
О: Зарегистрируйся на cloud.ru → AI Models → Создать ключ.

**В: Сколько стоит использование?**  
О: Зависит от тарифа cloud.ru. Примерный расход: 500–2000 токенов на задачу.

**В: Можно ли использовать другую модель?**  
О: Да, измени MODEL_NAME в файле .env на доступную в cloud.ru.

**В: Где сохраняются результаты?**  
О: В файле teamAi_report.md в корне проекта (автоматически).

**В: Как очистить историю задач?**  
О: Перезапусти веб-сервер — история хранится в памяти сессии.

**В: Можно ли запустить на удалённом сервере?**  
О: Да, в web.py измени host='0.0.0.0' и настрой firewall.

**В: Что делать при ошибке импорта?**  
О: Убедись, что запускаешь из папки проекта и установлены зависимости: pip install -r requirements.txt

**В: Почему Оркестратор не выбирает агентов?**  
О: Проверь, что модель GLM-4.7 поддерживает JSON в ответе. Как фолбэк — система подключает coder+reviewer по умолчанию.

## 🛠️ Разработка

### Добавить нового агента

1. Открой agents.py
2. Добавь нового агента в __init__:

```python
"new_agent": Agent(
    "Имя", "Роль",
    "Системный промпт...",
    "🆔"
)
```

3. Обнови выбор в веб-интерфейсе (templates/index.html)

### Изменить промпт агента

Открой agents.py → найди агента → отредактируй system_prompt.

### Добавить новую команду

1. Создай функцию в web.py с декоратором @app.route()
2. Обработай запрос и верни JSON
3. Вызови из JavaScript в index.html
