## 🎯 О проекте

**TeamAi** — это консольная система управления командой специализированных ИИ-агентов. Система принимает задачи от пользователя, делегирует их соответствующим моделям через API Cloud.ru Foundation Models и автоматизирует рутинные операции: написание кода, ревью, тестирование и интеграцию с GitHub.

### Ключевые возможности

| Функция | Описание |
|--------|----------|
| 🧠 **Оркестрация** | Умное распределение задач между агентами |
| 💻 **Кодинг** | Генерация Lua-кода для Reign RP (figura-fix) |
| 🔍 **Ревью** | Автоматическая проверка кода на ошибки |
| 🧪 **Тестирование** | Поиск уязвимостей и багов |
| 🔄 **GitHub** | Ветки, коммиты, пуши, Pull Request'ы |
| 🛡️ **Безопасность** | Подтверждение критических операций |
| 🎨 **TUI** | Красивый интерфейс в терминале (rich) |

## 📦 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/<your-username>/TeamAi.git
cd TeamAi
```

### 2. Установка зависимостей (системно)

```bash
# Вариант 1: Для текущего пользователя (рекомендуется)
pip install --user -r requirements.txt

# Вариант 2: Системная установка (требует sudo)
sudo pip install -r requirements.txt
```

### 3. Файл зависимостей `requirements.txt`

```txt
rich>=13.0.0
PyGithub>=2.0.0
python-dotenv>=1.0.0
requests>=2.31.0
```

## ⚙️ Конфигурация

### 1. Создание файла `.env`

```bash
cp .env.example .env
```

### 2. Заполнение переменных

```env
# Cloud.ru API (обязательно)
CLOUD_RU_API_KEY=your_cloud_ru_api_key_here

# GitHub (опционально, для API-интеграции)
GITHUB_TOKEN=your_personal_access_token
REPO_OWNER=your-github-username
REPO_NAME=TeamAi
DEFAULT_BRANCH=main

# Настройки
LOG_LEVEL=INFO
SAFE_MODE=true
```

### 3. Получение API-ключей

#### Cloud.ru Foundation Models
1. Зайдите в [консоль Cloud.ru](https://console.cloud.ru)
2. Перейдите в раздел **Foundation Models**
3. Создайте API-ключ в разделе **Access Tokens**

#### GitHub Token (если нужен)
1. GitHub → **Settings** → **Developer settings** → **Personal access tokens**
2. Создайте токен с правами:
   - `public_repo` (для публичных репозиториев)
   - `repo` (для приватных репозиториев)

## 🤖 Агенты

| Агент | Модель | Цвет | Специализация |
|-------|--------|------|---------------|
| **Оркестратор** | `zai-org/GLM-4.7` | 🟡 Gold | Распределение задач, анализ |
| **Кодер** | `Qwen/Qwen3-Coder-Next` | 🔵 Cyan | Lua, Reign RP, figura-fix |
| **Ревьюер** | `zai-org/GLM-4.7` | 🟣 Magenta | Code review, оптимизация |
| **Тестировщик** | `zai-org/GLM-4.7-Flash` | 🟢 Green | Поиск багов, уязвимости |

### Тег делегирования

Оркестратор использует специальные теги для вызова агентов:

```
[AGENT_CALL: CODER] — передать задачу Кодеру
[AGENT_CALL: REVIEWER] — отправить на ревью
[AGENT_CALL: TESTER] — запустить тестирование
```

## 🛡️ Безопасность

### Безопасный режим (Safe Mode)

По умолчанию включён. Требует подтверждения Сэра перед:

- ❗ Пушем изменений на GitHub
- ❗ Удалением файлов
- ❗ Изменением конфигурации

### Проверка путей

Система блокирует опасные операции:

```python
# Запрещённые паттерны
DANGEROUS_PATTERNS = ["../", "/etc/", "/root/", "rm -rf"]
```

### Логирование

Все действия записываются в `logs/teamai.log` для аудита.

## 🔌 API Cloud.ru

### Базовый URL

```
https://foundation-models.api.cloud.ru
```

### Эндпоинт чата

```
POST /chat/completions
```

### Пример запроса

```json
{
  "model": "zai-org/GLM-4.7",
  "messages": [
    {"role": "system", "content": "Ты полезный ассистент."},
    {"role": "user", "content": "Привет!"}
  ],
  "temperature": 0.2,
  "max_tokens": 4096
}
```

### Лимиты

| Параметр | Значение |
|----------|----------|
| Max tokens | 4096 (на запрос) |
| Rate limit | Зависит от тарифа |
| Timeout | 120 секунд |
